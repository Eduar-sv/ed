BACHILLERATO TÉCNICO VOCACIONAL EN DESARROLLO DE SOFTWARE

Descriptor del módulo de Tercer año

Módulo 3.5:“Desarrollo de componentes para dispositivos móviles”

Aspectos generales.
Campo: Desarrollo de Software
Opcion Desarrollo de Software
Competencia Desarrollar componentes de software para dispositivos móviles.
Titulo del modulo Desarrollo de componentes para dispositivos móviles.
Duración prevista seis semanas (180 horas).
Prerrequesito: Segundo año de bachillerato Código: BTVDS 3.5
Objetivo del módulo:

Desarrollar la competencia para construir soluciones y componentes de software, utilizando tecnologías de programación orientada a objetos con Java Micro Edition para dispositivos móviles.
Situación problemática:

El estancamiento de la expansión de las operaciones mercantiles hacia el sector de usuarios de dispositivos móviles, se debe a que no se cuenta con el desarrollo de componentes de software que operen en estos entornos de manera efectiva, generando baja competitividad y falta de presencia en estos mercados.
Criterios de evaluación:

1. Elabora el documento de especificación funcional y no funcional, utilizando las estrategias y artefactos del proceso unificado de desarrollo de software.

2. Elabora el documento de componentes de la arquitectura de hardware, OS y Firmware, haciendo uso de las especificaciones arquitectónicas del dispositivo, para identificar requerimientos y limitaciones, así como para adquirir el conocimiento de la arquitectura en la cual se desarrollará el componente.

3. Elabora el documento de la arquitectura de software, utilizando el documento de componentes de la arquitectura de hardware, OS y Firmware, así como las interfaces a utilizar para la persistencia de la información.

4. Elabora la actualización del documento de arquitectura de software, utilizando los componentes del framework seleccionado y los estándares de la industria para el desarrollo en componentes móviles, como los especificados en la versión Micro Edition de Java.

5. Desarrolla los componentes de código utilizando el documento de arquitectura y los estándares de la industria como el proceso unificado para el desarrollo de software.

6. Ejecuta baterías de pruebas utilizando las mejores estrategias que permitan asegurar el correcto comportamiento del componente de software. Esto perfectamente se puede hacer mediante herramientas de simulación.

7. Elabora la documentación de usuario y técnica, usando las especificaciones dictadas en el documento de arquitectura, un software ofimático, la operación funcional de la aplicación y la información relativa al mantenimiento de los componentes de software.

Criterio de promoción:

Comprobar que se ha alcanzado al menos el 70% de las competencias esperadas en una escala estimativa correspondiente a 7: nivel 4.

    Sugerencias metodológicas

    El docente debe presentar los aspectos sobre los que trata el módulo, especialmente los siguientes conocimientos tecnológicos: desarrollo de componentes para equipos móviles haciendo uso del proceso unificado de desarrollo de software y su vinculación con las oportunidades de empleabilidad. En esta etapa el docente debe organizar los equipos de trabajo para desarrollar las competencias del módulo. El facilitador debe orientar al grupo de estudiantes durante todas las etapas del módulo, para identificar aquellas actividades relacionadas con el desarrollo de componentes para dispositivos móviles.
